# 🏥 Sistema Omnichannel para Clínica Médica

[cite_start]Este projeto apresenta o desenvolvimento de um protótipo funcional de uma plataforma **Omnichannel** voltada para clínicas médicas, integrando múltiplos canais de comunicação com um banco de dados centralizado[cite: 319, 325].

[cite_start]O trabalho foi desenvolvido para a matéria de **Low-Code** na **UNIC** e foca na viabilidade da centralização de dados e na agilidade da prototipagem[cite: 324, 333, 334].

## 🛠️ Tecnologias e Ferramentas
* [cite_start]**Linguagens:** PHP e SQL[cite: 319, 365].
* [cite_start]**Banco de Dados:** MySQL Workbench[cite: 326].
* [cite_start]**Ambiente Local:** XAMPP (Apache e MySQL)[cite: 326, 351].
* [cite_start]**Editor:** VS Code[cite: 326, 353].
* [cite_start]**Abordagem:** Híbrida (Tradicional + Low-Code assistido por IA)[cite: 327, 343].

## 🏗️ Arquitetura do Sistema
[cite_start]O sistema utiliza uma estrutura de dados robusta para garantir a fluidez do atendimento[cite: 333, 345]:
* [cite_start]**Tabela de Pacientes:** Registra o canal preferido de contato[cite: 348].
* [cite_start]**Controle de Acesso:** Perfis distintos para Atendentes, Gerentes e Pacientes[cite: 349, 354].
* [cite_start]**Histórico Unificado:** Tabela de conversas que permite transferir atendimentos entre canais sem perda de contexto[cite: 350].

## 📊 Análise de Metodologia
[cite_start]O estudo reflete sobre a complexidade da integração entre plataformas de baixo código (como Bubble e Base44) e bancos de dados SQL externos, concluindo que o conhecimento em lógica de backend continua essencial para a escalabilidade[cite: 328, 360, 362].
